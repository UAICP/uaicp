# v0.5 Roadmap

## Scope

`v0.5` is the ecosystem and scale release. It builds on `v0.4` by improving interoperability, operations, and community growth.

## Planned Outcomes

- compliance badges generated from conformance CI artifacts
- pluggable verifier registry extensions (remote and signed verifier packs)
- richer observability integrations beyond OpenTelemetry baseline
- expanded policy profile catalog for different operational postures
- stronger governance process for multi-team and community participation

## Milestones

### M1: Compliance Certification UX

- deliverables:
- badge generation pipeline for official and community adapters
- public conformance summary page per adapter and version
- acceptance criteria:
- badge data is derived only from reproducible CI reports
- stale or unverifiable badges are automatically revoked

### M2: Verifier Registry Extensions

- deliverables:
- optional remote verifier discovery protocol
- signature and integrity checks for verifier packages
- acceptance criteria:
- adapters can load verifiers from local and remote sources with the same contract
- failed signature or trust checks hard-fail verifier loading

### M3: Observability Integrations

- deliverables:
- reference exporters and guides for Datadog and additional platforms
- SLO-aligned telemetry recommendations for gate latency and block rates
- acceptance criteria:
- reference dashboards show verifier/policy outcomes linked to traces
- documented minimum telemetry set validated in integration tests

### M4: Policy Profiles + Governance Scale

- deliverables:
- standardized profiles such as `strict`, `balanced`, and `expedite`
- formal proposal lifecycle (draft, review, accepted, deprecated)
- acceptance criteria:
- profile fixture packs pass across official adapters
- governance process supports external contributors with clear decision records

## Release Gates

- all `v0.4` guarantees remain intact
- profile and registry features have backwards-compatible defaults
- operational docs include rollout and rollback guidance for every new optional feature
