# UAICP Governance

UAICP is governed as a practical open-source specification project.

## Governance Principles

- reliability over hype
- neutral protocol contract
- transparent change process
- compatibility-aware versioning

## Decision Process

1. proposal issue
2. draft text + schema updates
3. review from maintainers and contributors
4. merge with migration notes when needed

## Maintainer Responsibilities

- protect contract stability
- enforce normative/informative distinction
- require evidence for behavioral claims
- prevent vendor-specific lock-in in core spec
